(home page)

Explanation

BlocItOff

Summary

A self-destructing to-do list application

(home page)

Explanation

To-do lists are notorious for collecting junk: to-do items that you want to remember, but are not very important and thus get consistently put off. To address the problem of to-do list clutter, I created Blocitoff. Blocitoff will aim to keep to-do lists manageable by automatically deleting to-do items that have not been completed after seven days. The hypothesis is that if the to-do item is not important enough to be completed in seven days, it doesn't belong on your to-do list

(to-do list page)

Conclusion

Users can easily sign in or up and immediately start creating their to-do lists. After an item is 7 days old, the item will automatically delete itself. Creating this project taught me more about the functionality ruby and ruby-on-rails has to offer; learning things such as creating my own "namespaces" and "tasks" along with an asortment of built in methods that help with time measurements.

Bloc Time

Summary

A time-management system based on the Pomodoro technique using AngularJS and Firebase

(home page)

Explanation

The Pomodoro Technique aims to increase your work productivity and quality. In its simplest form, you time your work in 25-minute intervals, with five-minute breaks in between.

During a 25-minute session, you focus on a task. Checking email, answering the phone, or browsing your favorite forum is not allowed. During a break, you do not do anything related to work. You can meditate, do push ups and play with your dog, but you can't think about work.

When implemented with discipline, the Pomodoro Technique can be a powerful tool to increase your productivity. I built an application to manage Pomodoro sessions

(firebase data)

Conclusion

After each 25 minute work session, a user will begin a 5 minute break. After every 4 work sessions, a break will consist of 30 minutes. Sessions will completly start over if the user manually stops the clock. A ding sound will play each time the clock reaches 0, alerting the user to take a break or start their next session. Data transverses through firebase and can manually be reset or edited via admin functionality.

  • Github: Try It Out or View the Code
  • Date: September 2016
  • Services: JS, Angular, Firebase, HTML, CSS, Bootstrap

Chat It Up

Summary

A real-time chat client using AngularJS and Firebase

(home page)

Explanation

I was interested in how a chat-room operated and learned the basics by creating this app. Utilizing Firebase's service, I wanted to give the ability to create multiple chat-rooms capable of using multiple users.

(firebase data)

Conclusion

This application was difficult to hit the ground running. Firebase was new to me, therefore I needed to learn its basic functionality so I could use it properly. Many times I had my data in the back-end populating wrong, but quickly found out it was because of my AngularJS routes. Once I fixed a few bugs, the rest of the application was fun to program. If a user navigates to the application without a "username" the application will automatically prompt for creation of one. After a user is succesfully logged in, they can enter a current chat room or create one of their own. Messages in each chat room do not exceed 30 messages and are displayed in current order from top down.
